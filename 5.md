## 0. CSS3 Syntax

**CSS3** is the current standard of CSS. Every CSS rule follows the same structure:

```
selector {
    property: value;
    property: value;
}
```

- **Selector** — which HTML element(s) to style
- **Property** — what aspect to change (color, size, spacing…)
- **Value** — the setting for that property
- Each `property: value` pair ends with a **semicolon `;`**
- The entire block is wrapped in **curly braces `{}`**
- Comments are written with `/* your comment */`

```css
/* This is a CSS comment */
h1 {
    color: navy;        /* property: value */
    font-size: 32px;
    text-align: center;
}
```

## 1. Three Ways to Add CSS

### Method 1 — Inline Style (directly on the element)

```html
<p style="color: red; font-size: 18px;">Hello World</p>
```

- Highest priority (overrides everything else)
- Hard to maintain — avoid using it for more than quick tests

### Method 2 — Internal Style Sheet (`<style>` inside `<head>`)

```html
<head>
    <style>
        p {
            color: red;
            font-size: 18px;
        }
    </style>
</head>
```

- Good for single-page projects (what we used in lesson 4)

### Method 3 — External Style Sheet (separate `.css` file) ✅ Best Practice

**`index.html`**
```html
<head>
    <link rel="stylesheet" href="style.css" />
</head>
```

**`style.css`** (a separate file in the same folder)
```css
p {
    color: red;
    font-size: 18px;
}
```

> **Why external?** One `.css` file can style **many** HTML pages at once.
> Change the CSS file → all pages update instantly.

| Method | Where written | Best for |
|---|---|---|
| Inline | `style=""` attribute | Quick one-off overrides |
| Internal | `<style>` in `<head>` | Single-page experiments |
| External | Separate `.css` file | Real projects ✅ |

## 2. CSS Selectors (Expanded)

### ID vs Class — Key Difference

| | ID `#name` | Class `.name` |
|---|---|---|
| Symbol | `#` | `.` |
| How many per page | **Once** (unique) | **Many times** |
| How many per element | One | Multiple allowed |
| Use for | A unique landmark (page title, main nav) | Reusable styles (cards, buttons) |

```html
<!-- ID: used once -->
<h1 id="site-title">My Website</h1>

<!-- Class: used many times -->
<p class="highlight">First paragraph</p>
<p class="highlight">Second paragraph</p>
```

```css
#site-title  { font-size: 40px; color: #2c3e50; }
.highlight   { background-color: yellow; }
```

An element can also have **both** an id and a class:
```html
<p id="intro" class="highlight">Hello</p>
```

### Element Selector

Targets all elements of that tag name.

```css
p    { color: #333; }
h1   { font-size: 36px; }
body { background-color: #f0f0f0; }
```

### ID Selector — `#id`

```html
<h2 id="main-title">Welcome</h2>
```

```css
#main-title {
    color: darkblue;
    text-align: center;
}
```

### Class Selector — `.class`

```html
<p class="warning">Do not delete this file.</p>
<p class="warning">This action is irreversible.</p>
```

```css
.warning {
    color: red;
    font-weight: bold;
    background-color: #fff3cd;
}
```

### Descendant Selector

Target an element **only when it is inside** another element.

```css
/* Only <p> tags that are inside a <div> */
div p {
    color: green;
}
```

```html
<div>
    <p>This text is green.</p>   <!-- targeted -->
</div>
<p>This text is NOT green.</p>  <!-- NOT targeted -->
```

---

### Multiple Selectors (Grouping)

Apply the same style to several elements at once using a comma `,`.

```css
h1, h2, h3 {
    font-family: Arial, sans-serif;
    color: #333333;
}
```

## 2. CSS Colors

### Named Colors
```css
color: red;
color: tomato;
color: steelblue;
```

### Hex Colors (`#RRGGBB`)
```css
color: #ff0000;       /* red */
color: #00ff00;       /* green */
color: #0000ff;       /* blue */
color: #333333;       /* dark gray */
color: #ffffff;       /* white */
color: #000000;       /* black */
```

### RGB Colors
```css
color: rgb(255, 0, 0);      /* red */
color: rgb(0, 128, 255);    /* bright blue */
color: rgb(100, 100, 100);  /* gray */
```

### RGBA (with transparency)
The 4th value `a` is **opacity**: `0` = fully transparent, `1` = fully visible.

```css
background-color: rgba(0, 0, 255, 0.3);   /* light transparent blue */
background-color: rgba(0, 0, 0, 0.5);     /* semi-transparent black */
```

## 3. The CSS Box Model

Every HTML element is a **box** made of four layers (from inside to outside):

```
+------------------------------------------+
|              MARGIN (outside)            |
|   +----------------------------------+   |
|   |          BORDER                 |   |
|   |   +--------------------------+  |   |
|   |   |       PADDING            |  |   |
|   |   |   +------------------+   |  |   |
|   |   |   |    CONTENT       |   |  |   |
|   |   |   +------------------+   |  |   |
|   |   +--------------------------+  |   |
|   +----------------------------------+   |
+------------------------------------------+
```

### `padding` — Space *inside* the border

```css
p {
    padding: 20px;              /* all 4 sides */
    padding: 10px 20px;         /* top/bottom  left/right */
    padding: 5px 10px 15px 20px; /* top right bottom left */
}
```

### `margin` — Space *outside* the border

```css
p {
    margin: 30px;         /* all 4 sides */
    margin: 10px 0;       /* top/bottom  left/right */
    margin: 0 auto;       /* center a block element horizontally */
}
```

### `width` and `height`

```css
div {
    width: 400px;
    height: 200px;
    background-color: lightgray;
}
```

You can also use percentages:

```css
div {
    width: 50%;   /* 50% of the parent element's width */
}
```

## 4. Borders (Deep Dive)

The `border` shorthand sets **width**, **style**, and **color** in one line:

```css
div { border: 2px solid black; }
```

### Border Styles

```css
border: 1px solid red;    /* solid line */
border: 2px dashed blue;  /* dashed line */
border: 3px dotted green; /* dotted line */
border: 4px double navy;  /* double line */
border: none;             /* no border */
```

### Individual Sides

```css
div {
    border-top: 3px solid red;
    border-right: 1px dashed blue;
    border-bottom: 3px solid red;
    border-left: none;
}
```

### `border-radius` — Rounded Corners

```css
div { border-radius: 10px; }   /* all corners */
div { border-radius: 50%; }    /* circle (if width = height) */
```

## 5. Block vs Inline Elements

HTML elements are either **block** or **inline** by default.

| Type | Behavior | Examples |
|---|---|---|
| **Block** | Takes up the full width, starts on a new line | `<div>`, `<p>`, `<h1>`–`<h6>`, `<ul>`, `<li>` |
| **Inline** | Takes only as much space as its content | `<span>`, `<a>`, `<strong>`, `<em>`, `<img>` |

### The `display` Property

You can override the default with `display`:

```css
span { display: block; }        /* make inline behave like block */
div  { display: inline; }       /* make block behave like inline */
div  { display: inline-block; } /* inline but respects width/height */
div  { display: none; }         /* hide the element completely */
```

### Practical example — buttons side by side

```html
<div class="btn">Save</div>
<div class="btn">Cancel</div>
```

```css
.btn {
    display: inline-block;
    padding: 10px 20px;
    background-color: steelblue;
    color: white;
    border-radius: 5px;
    margin: 5px;
}
```

> `inline-block` is your go-to when you want elements side by side **and** still control their `width`/`height`/`padding`.

## 6. CSS Text & Font Properties

### `font-family`

```css
p {
    font-family: Arial, sans-serif;
    font-family: "Times New Roman", serif;
    font-family: "Courier New", monospace;
}
```

> Always list a **generic family** (serif, sans-serif, monospace) as a fallback.

### `font-weight`

```css
p { font-weight: normal; }   /* default */
p { font-weight: bold; }
p { font-weight: 100; }      /* thin  (100 to 900) */
p { font-weight: 700; }      /* same as bold */
```

### `font-style`

```css
p { font-style: normal; }
p { font-style: italic; }
```

### `text-decoration`

```css
a  { text-decoration: none; }         /* remove underline from links */
p  { text-decoration: underline; }
p  { text-decoration: line-through; }
```

### `line-height`

Controls the vertical space between lines of text.

```css
p {
    line-height: 1.6;   /* 1.6× the font size — good for readability */
}
```

### `letter-spacing`

```css
h1 {
    letter-spacing: 3px;
}
```

### `text-transform`

```css
h1 { text-transform: uppercase; }
p  { text-transform: capitalize; }  /* First letter of each word */
```

## 8. Media Queries — Making Pages Responsive

A **media query** checks the screen size (or device type) and applies CSS only when the condition is true. This is how we build **responsive** websites.

```css
@media (max-width: 768px) {
    /* These styles apply ONLY on screens 768px wide or smaller */
    body {
        font-size: 14px;
    }
}
```

### Common Breakpoints

```css
/* Large screens — desktops */
@media (min-width: 1024px) { ... }

/* Medium screens — tablets */
@media (max-width: 1023px) { ... }

/* Small screens — phones */
@media (max-width: 600px) { ... }
```

### Example — hide a sidebar on phones

```css
.sidebar {
    display: block;   /* visible on desktop */
}

@media (max-width: 600px) {
    .sidebar {
        display: none;   /* hidden on phone */
    }
}
```

### Example — change layout on small screens

```css
.container {
    width: 80%;
    margin: 0 auto;
}

@media (max-width: 600px) {
    .container {
        width: 100%;
        padding: 10px;
    }
}
```

> **Mobile-first tip:** write your default CSS for phones, then use `min-width` queries to enhance for larger screens.

## 9. CSS Grid Layout

**CSS Grid** is the most powerful layout system in CSS. It lets you design pages in **rows and columns** without using `<table>`.

### Setting Up a Grid

```html
<div class="grid-container">
    <div>Cell 1</div>
    <div>Cell 2</div>
    <div>Cell 3</div>
    <div>Cell 4</div>
    <div>Cell 5</div>
    <div>Cell 6</div>
</div>
```

```css
.grid-container {
    display: grid;
    grid-template-columns: 1fr 1fr 1fr;  /* 3 equal columns */
    gap: 20px;                           /* space between cells */
}
```

`1fr` means **1 fraction** of the available space. Three `1fr` columns = three equal columns.

### Different Column Widths

```css
/* 3 columns: small, large, small */
grid-template-columns: 1fr 2fr 1fr;

/* Fixed + flexible */
grid-template-columns: 200px 1fr;

/* Repeat shorthand */
grid-template-columns: repeat(3, 1fr);   /* same as 1fr 1fr 1fr */
grid-template-columns: repeat(4, 1fr);   /* 4 equal columns */
```

### Rows

```css
.grid-container {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    grid-template-rows: 100px auto 100px;  /* header, content, footer */
    gap: 15px;
}
```

### Spanning Cells

Make one cell stretch across multiple columns or rows:

```css
.wide-cell {
    grid-column: span 2;   /* takes up 2 columns */
}

.tall-cell {
    grid-row: span 2;      /* takes up 2 rows */
}
```

### Responsive Grid — auto-fit

This is the most useful pattern for real projects:

```css
.grid-container {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
    gap: 20px;
}
```

- **`auto-fit`** — fit as many columns as possible
- **`minmax(200px, 1fr)`** — each column is at least 200px, up to 1fr
- On a wide screen → many columns; on a phone → 1 column. **No media query needed!**

### Full Grid Page Example

```html
<!DOCTYPE html>
<html>
<head>
    <style>

        * { box-sizing: border-box; margin: 0; padding: 0; }

        body {
            font-family: Arial, sans-serif;
            background-color: #f0f0f0;
        }

        /* --- Page layout grid --- */
        .page {
            display: grid;
            grid-template-columns: 1fr;
            grid-template-rows: auto 1fr auto;
            min-height: 100vh;
        }

        header {
            background-color: #2c3e50;
            color: white;
            padding: 20px;
            text-align: center;
        }

        footer {
            background-color: #2c3e50;
            color: white;
            padding: 15px;
            text-align: center;
        }

        /* --- Card grid inside main --- */
        .card-grid {
            display: grid;
            grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
            gap: 20px;
            padding: 30px;
        }

        .card {
            background-color: white;
            border: 1px solid #ddd;
            border-radius: 10px;
            padding: 20px;
            box-shadow: 2px 2px 8px rgba(0,0,0,0.1);
        }

        .card h3 { color: #e74c3c; margin-bottom: 10px; }
        .card p  { line-height: 1.6; color: #555; }

        /* --- Responsive: stack on small screens --- */
        @media (max-width: 480px) {
            .card-grid {
                grid-template-columns: 1fr;
                padding: 15px;
            }
        }

    </style>
</head>
<body>

    <div class="page">

        <header>
            <h1>My CSS Grid Page</h1>
        </header>

        <main class="card-grid">
            <div class="card">
                <h3>HTML</h3>
                <p>Structure of the web. Tags, elements, and attributes.</p>
            </div>
            <div class="card">
                <h3>CSS</h3>
                <p>Styling the web. Colors, layouts, and animations.</p>
            </div>
            <div class="card">
                <h3>JavaScript</h3>
                <p>Behaviour of the web. Events, logic, and interaction.</p>
            </div>
            <div class="card">
                <h3>Responsive</h3>
                <p>Websites that look good on every screen size.</p>
            </div>
        </main>

        <footer>
            <p>&copy; 2026 Web Design Course</p>
        </footer>

    </div>

</body>
</html>
```
